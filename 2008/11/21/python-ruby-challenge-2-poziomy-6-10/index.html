Gist - odpicowane pastie

01 Sty 2009

Gist istnieje już jakiś czas, dopiero teraz przyglądnąłem się mu bliżej. Początkowo wydawało mi się, że to kolejny klon pastie, ale szybko okazało się, że to coś więcej.

Coś więcej ponieważ Gist, prócz tego co potrafi Pastie (edycja pliku online, kolorowanie składni, prywatne pliki), dodaje to co potrafi najlepiej - obsługa repozytoriów Git. Otóż każde nowe "pastie" powoduje założenie repozytorium gita, które następnie możemy pobrać (clone), a także wysyłać do niego zmiany (push). Oczywiście nie jesteśmy ograniczeni do 1 pliku, możemy mieć ich tyle ile chcemy. Na stronie pokazywana jest historia rewizji. Dbałość o szczegóły jest firmowym znakiem serwisu GitHub. Żegnaj Pastie, niech żyje Gist!

Python (Ruby) Challenge #2, poziomy 6-10

5 komentarzy | Kategorie: Python, Ruby, Techblog | trackback
Tagi: challenge game programming python riddles ruby

Jakiś czas temu starałem się zachęcić Was do zabawy pod nazwą "Python Challenge". Umieściłem wtedy rozwiązania pierwszych 5 zagadek. Brak większego odzewu spowodował, że do tej pory nie opisałem rozwiązań kolejnych zagadek. Uważam jednak, że temat jest wart odświeżenia.

Jeśli jeszcze nie wiesz jaka jest idea tej zabawy to opiszę Ci ją w kilku zdaniach. Idea jest następująca. Zabawę zaczynasz od strony startowej. Twoim celem jest odgadnięcie adresu następnego etapu. Podpowiedzi należy szukać w obrazku widocznym na stronie jak i w samym jej źródle. Przykładowo w zagadce oznaczonej numerem zero widać monitor, na którym jest napis "238". Jeśli obliczymy wartość tego wyrażenia a następnie podmienimy w adresie URL 0 na tą wartość dostaniemy się do następnego etapu. Następny poziomy są co raz trudniejsze.

Chociaż strona sugeruje, że etapy należy pokonywać przy pomocy Pythona to w praktyce możemy użyć dowolnego języka/narzędzia. Tylko 2 (lub coś koło tego) zagadki wymagają Pythona (ze względu na specyficzne moduły, np do serializacji obiektów). Dodam tylko, że osobiście uważam, że taki rodzaj zabawy jest także świetnym sposobem na naukę języka.

Rozwiązania poziomów 6-10

Uwaga! Poniżej zamieszczam rozwiązania kolejnych poziomów (6-10). Nie czytaj dalej jeśli nie chcesz psuć sobie zabawy.

Poziom 6

Otwierając źródło w pierwszej linijce widzimy: 

<html> <!-- <-- zip -->
...

Strzałka wskazuje na "html", co sugeruje by zamienić końcówkę adresu URL z "html" na "zip". Plik channel.zip pobieramy na dysk. Otwierając plik możemy zauważyć, że archiwum zawiera pliki o nazwach będących numerami. Zaglądamy do którego z plików i widzimy mniej więcej taką treść: 

Next nothing is 57831

Co oczywiście powinno przypominać nam jeden z wcześniejszych poziomów. Jeśli dobrze się przyjrzymy to zauważymy także plik readme.txt o zawartości: 

welcome to my zipped list.

hint1: start from 90052
hint2: answer is inside the zip

Pierwsza myśl podpowiada nam by rozpakować archiwum i zaczynając od pliku 90052.txt dotrzeć do takiego, który nie będzie zawiera informacji o następnym. Niestety tak znaleziony plik zawiera tylko: 

Collect the comments.

Okazuje się, że do plików w archiwum zip można dodawać notatki (komentarze). I to w nich znajduje się rozwiązanie zagadki. Ja skorzystałem z gema zipruby. 

require 'rubygems'
require 'zipruby'

file = "90052.txt"; out = ""
Zip::Archive.open('channel.zip') do |zip|
  loop do
    zf = zip.fopen(file)
    out << zf.comment
    file = zf.read.match(/Next nothing is (\d+)/)[1] + ".txt" rescue break
    zf.close
  end
end

Poziom 7

Szybkie spojrzenie na obrazek i możemy zauważyć, że:
na obrazku znajduje się pasek w odcieniach szarości
pasek zaczyna się od piksela 0 (współrzędna y), a kończy na 607
pasek podzielony jest na regularne bloki o rozmiarach 7x9 pikseli (prócz pierwszego bloku)
Ponieważ kolory szare mają taką samą wartość dla kanałów RGB oraz ich wartości (dla obrazków 24bit) są z przedziału 0-255 to możemy przypuszczać, że kolejne bloki reprezentują znaki ASCII. 

require 'rubygems'
require 'open-uri'
require 'RMagick'

image = Magick::Image.read('http://www.pythonchallenge.com/pc/def/oxygen.png')[0]
x, y = 0, 50
while x <= 607
  print((image.pixel_color(x, y).red >> (Magick::QuantumDepth - 8)).chr)
  x += 7
end

Uruchamiając dostajemy: 

smart guy, you made it. the next level is [105, 110, 116, 101, 103, 114, 105, 116, 121]

Musimy więc jeszcze podobny zabieg zrobić z podanym ciągiem liczb: 

#smart guy, you made it. the next level is [105, 110, 116, 101, 103, 114, 105, 116, 121]
puts [105, 110, 116, 101, 103, 114, 105, 116, 121].map {|e| e.chr}.join

integrity

def bunzip2(binary)
  open("|bunzip2", "r+") do |cmd|
    cmd.write(binary)
    cmd.close_write
    puts cmd.gets
  end
end

un = "BZh91AY&SYA\xaf\x82\r\x00\x00\x01\x01\x80\x02\xc0\x02\x00 \x00!\x9ah3M\x07<]\xc9\x14\xe1BA\x06\xbe\x084"

Poziom 9

Tytuł mówi "connect the dots". Jednak nie chodzi o punkty na obrazku (aczkolwiek obrazek jest kluczową podpowiedzią). W źródle znajdziemy ciąg liczb oznaczonych etykietami "first:" i "second:". Biorąc po 2 liczby należy rysować punkty. 

require 'rubygems'
require 'open-uri'
require 'RMagick'

content = open("http://www.pythonchallenge.com/pc/return/good.html", :http_basic_authentication => ["huge", "file"]).read
first = content[/first:\n(.*?)\n\n/m, 1].split(/,\n?/).map {|n| n.to_i }
second = content[/second:\n(.*?)\n\n/m, 1].split(/,\n?/).map {|n| n.to_i }

max_coordinate = (first + second).max
image = Magick::Image.new(max_coordinate, max_coordinate) do
  self.background_color = 'white'
end

draw = Magick::Draw.new
draw.fill('white')
draw.stroke('black')

draw.polygon(*first)
draw.polygon(*second)
draw.draw(image)

image.display

Poziom 10

Kliknięcie w byka na obrazku prowadzi do pliku sequence.txt o treści: 

a = [1, 11, 21, 1211, 111221,

Nasze zadanie to obliczyć len(a[30]), czyli długość 30 elementu tej sekwencji. Nie jest łatwo odgadnąć jak powstaje ten ciąg, ale od czego mamy google? Wpisanie tych elementów do wyszukiwarki doprowadzi nas do wyjaśnienia znaczenia tego ciągu. Oto jak powstają elementy tej sekwencji:
liczba - opis słowny
1 - jedna jedynka (11)
11 - dwie jedynki (21)
21 - jedna dwójka, jedna jedyna (1211)
1211 - jedna jedynka, jedna dwójka, dwie jedynki (111221)
itd

Ma to nawet swoją nazwę Look-and-say sequence (a także "Morris Number Sequence"). 

def look_and_say_sequence(n)
  return "1" if n == 0
  return look_and_say_sequence(n - 1).scan(/((.)(\2*))/).map{|m| "#{m[0].size}#{m[1]}"}.join
end

puts look_and_say_sequence(30).length

Szukane rozwiązanie: 

5808
